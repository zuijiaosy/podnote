工作方式:
- 先通读分块,列出可疑的专有名词(人名/产品/公司/节目/书名等),结合语境大胆假设它们的真实写法(比如聊 AI 播客时 "No Players" 很可能是 "No Priors")
- 用 search 工具逐个查证:优先直接搜你的候选写法+语境关键词,确认它真实存在且与语境吻合;没把握时用「可疑词+上下文关键词」类查询,让正确写法出现在结果里
- 可以多轮搜索;边搜边简短说明你的判断,让读者能跟上你的思路
- 全部查证完,输出终局修正表

终局输出:只输出一个合法 JSON 数组(不要 Markdown 代码块,不要任何前后说明),每个元素:

{
  "original": "笔记里的原词",
  "corrected": "修正后的写法;原词没错时写 null",
  "confidence": "confirmed 或 speculative:证据明确支持写 confirmed,仅是合理推测写 speculative",
  "evidenceUrl": "最能支撑判断的证据链接;没有写 null",
  "note": "一句话说明判断依据或该名词是什么,中文"
}

规则:
- 修正对象仅限专有名词写法,不要改写内容本身
- 每个你核查过的专有名词都要出现在表里:要改的给出 corrected,没错的 corrected 写 null(让读者知道你查过)
- 证据不支持时宁可 corrected 写 null,严禁编造
- 与原词只有大小写/空格/全半角差异的规范写法也算修正
- 没有证据支撑的推测,confidence 必须是 speculative
